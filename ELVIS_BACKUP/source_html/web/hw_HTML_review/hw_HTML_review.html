HTML Review

HTML nesting structure

  • containers must nest
  • parent elements contain child elements
  • the browser makes a tree of nodes based on the HTML
  1. Basic Syntax:
    • All elements (nodes in the final parsed tree) are identified by tags.
  2. Tag Format:
    • If the node consists of text (such as a paragraph or something) then the tag gets a closing tag:
      • <tag [attribute="value" . . .] >text content</tag>
    • If the node contains no text (such as an image), then there is no closing tag:
      • <tag [attribute="value" . . .] />
    • all tags inside angle brackets
    • all tag and attribute names in lowercase
    • attributes are name="value" (space allowed)
      • value is case-sensitive
      • line breaks in values should be avoided!
    • unknown tags or attributes are ignored
      • (So a misspelled attribute will act as if nothing was there; use HTML validator to find typos.)
    • attributes in any order; if omitted, default is used
      • (some attributes are required; there's no default image file for the "img" tag)
    • all elements accept some attributes, called "core" attributes:

Core Attributes

These are things which specify something about an element in a document.

id:
unique name for this element
style:
CSS instructions for this element
class:
CSS class name
title:
sometimes used for browser tool-tips

HTML Elements

These create a new element in the document's tree.

  1. Document-Level Elements (top level):
    • html
    • head
    • body
      • (body can contain only block-level elements)
  2. Elements that go in the head:
    title:
    what goes in the title bar
    base:
    sets an href for where to start with links
    style:
    CSS instructions for a page (override CSS instructions linked from outside sources)
    link:
    related documents such as favicons or style sheets
    meta:
    metadata (says something about the data, such as author)
    script:
    JavaScript (either in-page or external)
  3. Block-Level Elements (act like paragraphs; go in the body):
    p:
    paragraphs (cannot contain other blocks)
    h[1-6]:
    headings (think of an outline)
    pre:
    preformatted block (fixed-width font, no rejustification, but HTML links and such work)
    div:
    user-specified block
    hr:
    horizontal rule (this one is NOT a container!)
    lists:
    ul, ol:
    [un]ordered list (list items take "li")
    dl:
    definition list (list items are 'dt' and 'dd')
    table:
    makes a grid
    form:
    get info from user
    blockquote:
    what it says; usually indented (use <p> tags inside)
  4. Inline Elements (must be inside a block element):
    a:
    anchor (can be named, may include href)
    How you make links:
    1. Use the "href" attribute to specify what the browser should load when clicked.
      • Do not use "click here" as anchor text.
      • "jump links" let you go to another point on the same page, or go to a new page and skip down to a particular location.
    2. Set an anchor to mark the location:
      • <a name="foo" /> (not a container)
    3. Make the link using:
      • <a href="#foo">text</a> (same page)
      or:
      • <a href="URL#foo">text</a> (other page)
    em/strong:
    emphasis/strong emphasis (usually italic/bold)
    sub, sup:
    subscript/superscript
    code:
    computer code
    var:
    variable name
    kbd:
    text for user input
    samp:
    sample output
    span:
    user-defined section of text
    q:
    in-line quote (try '<q lang="en">' and "fr")
    cite:
    citations
    dfn:
    definitions
    var:
    variables; usually italics
    characters:
    HTML Character Entities; note "&shy;" for soft hyphen, if you have some really long words that it's okay to break.
    figure:
    container for one or more images with a caption
    figcaption:
    caption for an image or group of images
    non-container elements:
    br:
    line break
    img:
    pictures (more later)
    Appearance/physical tags (avoid: use CSS instead):
    • b
    • i
    • center
    • font
    • big
    • small
    • u
    • tt
    • strike
